# 贡献指南

欢迎来到“文明演化通解-特解模型”的思想共同体！

您对本项目的任何关注，本身就是对“合作优于孤立”这一“特解”理念的践行。我们坚信，深刻的思想不应被禁锢于个体的大脑，而应在开放、理性的碰撞中生长。本指南旨在邀请您，与我们共同雕琢这一理论。

## 🧠 贡献理念：从“黑暗森林”到“协作绿洲”

在本项目中，我们共同实践从“认知通解”到“认知特解”的跃迁：
- **我们反对**：动机揣测、人身攻击、立场优先的“认知黑暗森林”。
- **我们倡导**：就事论事、逻辑严谨、尊重事实的“理性协作绿洲”。

您的每一次贡献，都是在为这片珍贵的绿洲添砖加瓦。

## 🌟 如何贡献

我们欢迎任何形式的贡献，无论大小。

### 1. 提出议题与讨论
如果您发现任何问题或有新的想法，最直接的方式是在 [GitHub Issues](链接到您的仓库的Issues页面) 中发起讨论。我们特别欢迎以下类型的议题：
- **逻辑漏洞**：指出理论体系中任何不自洽、不严谨之处。
- **概念澄清**：对某些公理或概念提出疑问，要求更清晰的阐述。
- **应用拓展**：提供新的案例（历史的、科幻的、当下热点的）来验证或挑战模型。
- **跨界连接**：建议可以连接物理学、复杂科学、生物学、经济学等领域的相关理论。

### 2. 直接贡献内容
如果您希望直接修改或增补文档，请通过 **Pull Request** 的方式提交。
**工作流程**：
1.  **Fork** 本仓库到您的账户下。
2.  在您的仓库中创建一个新的分支（例如：`patch-theory-logic`）。
3.  在您的新分支上进行修改。
4.  提交更改，并推送（Push）到您的远程仓库。
5.  在本仓库发起 **Pull Request**，并清晰说明您的修改内容和理由。

我们欢迎的内容贡献包括但不限于：
- **修正错别字与语法错误**。
- **润色表述，提升文档的可读性**。
- **增补理论的推论或阐释**。
- **撰写基于本理论的科普文章、分析案例**（可放置于 `examples/` 目录）。
- **将理论翻译成其他语言**（可放置于 `i18n/` 目录）。

### 3. 传播与衍生创作
您可以通过以下方式，在更广阔的世界里贡献本项目的影响力：
- **分享链接**：在合适的论坛、社区或社交媒体分享本项目。
- **创作衍生作品**：基于本理论进行小说、文章、视频、艺术等创作。
- **教学与应用**：在课程、讲座或战略分析中引用和应用此模型。

**请注意**：所有衍生作品需遵守本项目的 **CC BY-SA 4.0** 许可证。这意味着您的作品也必须在相同条款下开放，并为您的工作署名。

## 📝 贡献规范

为确保协作的顺畅与高效，请遵循以下规范：

- **保持理性与尊重**：我们讨论的是思想，而非攻击个人。请用逻辑和证据支持您的观点。
- **引用与溯源**：当您引用外部资料或他人的观点时，请务必注明出处。
- **一次聚焦一个议题**：每个 Issue 或 Pull Request 应尽可能专注于解决一个问题，这有助于深入讨论和快速合并。
- **接受不确定性**：这是一个探索中的理论，并非终极真理。我们接受建设性的批评和健康的辩论。

## ❓ 寻求帮助

如果您对贡献流程有任何疑问，可以直接在 [Discussions](链接到您的仓库的Discussions页面，若开启) 或 [Issues](链接到Issues页面) 中提问，社区成员会尽力提供帮助。
